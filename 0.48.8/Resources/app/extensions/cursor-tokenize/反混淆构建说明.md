# cursor-tokenize 反混淆构建说明

本文档介绍如何使用反混淆后的源代码构建cursor-tokenize扩展的非混淆版本。

## 1. 项目结构

完成反混淆后，项目结构如下：

```
cursor-tokenize/
├── dist/                  # 编译后的文件
│   └── main.js            # 原始混淆版本
├── src/                   # 反混淆后的源代码
│   ├── extension.js       # 扩展入口点
│   ├── ControlProvider.js # 控制提供程序
│   ├── Tokenizer.js       # 标记化器实现
│   ├── tiktoken.js        # WebAssembly加载器
│   └── tiktoken_bg.js     # WebAssembly绑定
├── package.json           # 更新后的项目配置
├── webpack.config.js      # 构建配置
└── 反混淆分析报告.md       # 详细分析报告
```

## 2. 前置条件

构建项目需要以下工具：

- Node.js (建议 v14 或更高版本)
- npm (通常随Node.js一起安装)

## 3. 安装依赖

首先，需要安装项目依赖：

```bash
cd 0.48.8/Resources/app/extensions/cursor-tokenize
npm install
```

这将安装package.json中列出的所有依赖项，包括：

- @dqbd/tiktoken - 核心标记化库
- webpack - 用于构建项目
- babel - 用于JavaScript转换
- 其他开发工具

## 4. 构建项目

安装完依赖后，可以使用以下命令构建项目：

```bash
npm run webpack
```

或者，如果需要清理以前的构建：

```bash
npm run build-clean
```

构建完成后，将在`dist`目录中生成新的`main.js`文件，这是非混淆版本的扩展入口点。

## 5. 开发模式

如果需要在开发中实时查看变更，可以使用watch模式：

```bash
npm run watch
```

这将监视`src`目录中的文件变化，并在文件保存时自动重新构建项目。

## 6. 调试

为了调试扩展，可以使用以下步骤：

1. 构建开发版本：
   ```bash
   npm run webpack-dev
   ```

2. 使用VSCode运行和调试扩展（如果可用）

## 7. WebAssembly文件

注意，tiktoken_bg.wasm是一个二进制文件，不能直接通过文本编辑进行反混淆。反混淆后的代码中，我们只能改进其JavaScript包装器和绑定代码。

## 8. 验证构建

要验证构建是否成功，可以检查：

1. 确保dist/main.js文件已被更新
2. 文件大小应与原始文件相似，但不是压缩的
3. 如果可能，在Cursor中测试扩展功能

## 9. 常见问题

### Q: 构建时遇到"Cannot find module '@dqbd/tiktoken'"错误
A: 确保已正确安装依赖，可能需要手动安装: `npm install @dqbd/tiktoken`

### Q: WebAssembly模块加载失败
A: 确保tiktoken_bg.wasm文件位于正确位置。构建过程应该会复制此文件到dist目录。

### Q: 构建后的扩展无法激活
A: 检查package.json中的main字段是否正确指向"./dist/main.js"。

## 10. 结论

通过以上步骤，可以从反混淆的源代码构建出功能相同但更易读易维护的cursor-tokenize扩展。这有助于更好地理解扩展的工作原理，以及如何与Cursor的其他组件交互。
